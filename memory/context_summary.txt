# Research Assistant Project Context Summary

## Project Overview
The Research Assistant is a tool to help researchers search and analyze a collection of academic papers. It provides indexing and search capabilities for text-based documents.

## File Structure
See memory/file_structure.txt for complete listing

Key directories:
- research_assistant/: Main package containing cli.py, indexer.py, searcher.py
- test_papers/: Sample documents for testing
- memory/: Created for context preservation
- .claude/: Claude Code configuration

## Component Summaries

### research_assistant/__init__.py
Empty package initializer.

### research_assistant/cli.py
Command-line interface using argparse with two subcommands:
- `index`: Indexes documents from a directory (supports .txt, .pdf, .docx, .md)
- `search`: Searches the index using BM25 ranking algorithm

### research_assistant/indexer.py
Responsible for extracting text from files and building an inverted index:
- Supports .txt, .pdf (requires PyPDF2), .docx (requires python-docx), .md files
- Tokenization: lowercase, regex word boundary splitting, stopword removal (NLTK), Porter stemming (NLTK)
- Builds inverted index: term -> {doc_id: frequency}
- Saves index to index.json in the target directory

### research_assistant/searcher.py
Handles loading the index and performing searches:
- Loads index.json and computes document statistics (lengths, average length)
- Implements BM25 ranking algorithm for search relevance
- Tokenization matches indexer's approach (stopwords, stemming)
- Returns ranked list of (document_id, score) tuples

## Dependencies (requirements.txt)
- PyPDF2>=3.0.0: PDF text extraction
- nltk>=3.8: Natural Language Toolkit for stopwords and stemming

## Configuration (.claude/settings.local.json)
Contains PowerShell permission allowances for:
- Running foundation_test.py

- Executing research assistant skill driver
